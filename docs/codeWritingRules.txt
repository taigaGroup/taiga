1. Отступ для каждой новой строки 2 пробела (НЕ ИСПОЛЬЗОВАТЬ ТАБУЛЯЦИЮ)
2. Открываем новую скобу всегда в той же строке, что и оператор или функция и.т.п
3. Пробел перед каждой открытой скобкой
4. Добавлять краткое описание в шапке каждого макроса т.е описывать что он делает и кто автор (English)
5. Если создаются дополнительные функций, всегда писать кометарий перед имплементацией того, что эта функция делает
6. Строки не должны выходить за пределы экрана (без прокруток кода в право)
7. Временные переменные называть МАКСИМАЛЬНО понятно. (Без a, aa ,bb и.т.п)
8. Все переменные должны начинаться с маленькой буквы, но каждое новое слово в названий ВСЕГДА с большой! (например, timeOfLight)
9. Все имена динамических массивов должны начинаться с vectOf (например, vectOfEvents). Т.е. для массивов с нефиксированным размером
10. Имена статических массивов должны начинаться с arr. Т.е. для массивов с фиксированным размером
11. Целочисленная переменная в которую записывается какой-либо НОМЕР должна начинаться с большой буквы N (например,NEvent)
12. Целочисленная переменная, которая характеризует какое-либо количество чего-либо начинать с numberOf (например, numberOfEvents)
13. Каждый макрос в котором есть чтение файла, должен читать файлы ТОЛЬКО идентичной структуры!
14. Для основных типов данных ВСЕГДА использовать те что в ROOT (Double_t, Int_t и.т.п)
15. Пользоваться ТОЛЬКО константами ROOT (kTRUE, kFALSE и.т.п)
16. Новые константы доны быть объявлены в заголовочном файле, где описаны классы. Все символы с верхним реестром и разные слова в названий должны быть разделены нижним подчеркиванием (например, NUMBER_OF_PIXELS)
17. Названия макросов должны четко отражать то, что он делает
18. Названия классов с большой буквы
19. Названия функций с маленькой буквы
20. Названия встраиваемых классов всегда должно начинаться с префикса TAIGA. Например, class TAIGACamera.
21. Наследуем правила написания классов из ROOT
